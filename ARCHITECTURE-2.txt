Summary of changes applied for production gateway routing:

- Single gateway container (Nginx) handles all incoming HTTP requests and proxies to backend Django services.
- Routes added and normalized: requests to `/cms/`, `/tour/`, `/remittance/`, and `/certificates/` are explicitly redirected/normalized to trailing-slash URLs before proxying.

- Nginx forwards original browser headers to upstream services: `Host`, `X-Forwarded-Host`, and a custom `X-Tenant` header carrying tenant context. This preserves the original host and tenant info so tenant middleware can select the proper schema.
- CMS kept as the default fallback service at `/` while the explicit paths (`/cms/`, `/tour/`, `/remittance/`, `/certificates/`) are routed to their respective backends.

Actions taken to fix 502 Bad Gateway (applied to `gateway/local.conf`):

- Added resilient proxy settings: `proxy_connect_timeout 5s`, `proxy_send_timeout 60s`, `proxy_read_timeout 60s`, and `proxy_next_upstream error timeout invalid_header http_502 http_503 http_504` to allow upstream retries on transient errors.
- Kept trailing-slash behavior for prefix locations (`/cms/`, `/tour/`, `/remittance/`, `/certificates/`) so Nginx rewrites the path predictably when proxying to upstreams.
